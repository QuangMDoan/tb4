Two issues here:
1) Rate dropped from 30 Hz -> ~8 Hz again when YOLO + fusion are running 
    Bandwidth saturation from multiple image streams over WiFi

2) point_cloud_xyz_node can't sync 
    receives 0 image messages but gets camera_info (small messages flow, large images don't)

3) check:     ros2 topic info /oakd/stereo/image_raw -v 2>&1 
    See appendix 1

4) Root causes:
    - running the depth_to_pointcloud node on PC forces ~15 MB/s of raw depth images over WiFi. 
    - Combined with the RGB preview for YOLO -> saturating the link

5) Solutions 

    - Run point_cloud_xyz_node on the robot so depth-->pointcloud conversion happens locally 
        and only the smaller PointCloud2 flows over the network. 
        
    - Alternatively, use compressed image transport

6) Implementation (2026-04-04)
    Switch to compressed image transport in the launch file 
    - easier, keeps everything on Ubuntu PC

    IF "switch to compressed image transport" not working then:

    a) Move depth_to_pointcloud to the robot. depth stays local on Pi, only PointCloud2 goes over WiFi (~2 MB/s vs ~15 MB/s for raw depth images). 
        -Best for bandwidth but adds Pi CPU load
    
    b) Run everything on the robot. YOLO inference a Pi
